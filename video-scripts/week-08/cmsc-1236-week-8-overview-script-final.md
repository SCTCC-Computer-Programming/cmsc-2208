## What You're Doing This Week

This week's assignment is a module creation exercise. You receive a main program that imports a module, and you write that module. The main program defines the data, calls your functions, and displays the results. Your module is where all the logic lives.

The interface is fixed. Your function names, parameters, and return values have to match what the main program expects. What's different from previous application weeks is what happens inside each function. In Week 6, the instructions told you which concept to use for each function. This week, the assignment page describes what each function should accomplish, and you decide how to get there. You're working within a defined structure, but you have more responsibility for choosing the right approach.

That also means reading the main program matters more. The display functions show you what attributes and methods they access on your objects. The data constants show you what your functions will receive. The expected output shows you the target. Understanding that interface is how you figure out what your module needs to do. The one thing you do change in the main program is the import statement to match your filename, and the assignment page walks through exactly how.

The module has one class and six functions. Each one handles a different part of the game logic. The concepts come from Chapters 12 and 13, with some connections back to extended unpacking from Week 5 and class design from Week 6.
